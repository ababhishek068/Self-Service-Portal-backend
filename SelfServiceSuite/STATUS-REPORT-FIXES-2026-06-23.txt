STATUS REPORT IMPLEMENTATION - 23 JUNE 2026
===========================================

Implemented in this package
---------------------------
HR
- Leave supports decimal/half-day applications without sending numeric values to BC Boolean fields.
- Leave dates use BC-calculated end and return dates.
- Approval history is displayed in BC sequence order.
- Pending approval rows resolve full request details when the source record is available.
- Attendance captures and displays location coordinates.
- Training and Salary Advance retain draft, edit, request-approval and cancel-approval flows.
- Forgot password uses an email reset token; public self-registration is disabled.

Finance
- Imprest, Surrender, Staff Claim and Petty Cash use concise business detail screens.
- Relevant department, responsibility centre, employee account, place-of-duty and total fields
  are shown when BC returns them.
- Draft headers remain editable; request approval is available only after required lines exist.
- Cancellation is available only while an approval request is pending.
- Unsupported line editing is hidden; add/delete and surrender settlement actions remain available.
- Petty Cash shows the BC transaction name and amount for each line.

Facilities
- Purchase lines show Type, No., Description, Purpose, Quantity, Unit and Location.
- Store requisitions block backdated request dates and retain item/asset line handling.
- Purchase, Store, Transport, Fuel, Maintenance and Transfer details show only configured fields.
- Maintenance supports fixed-asset and vehicle-service request types, item, quantity, priority,
  location, problem description and the 5,000 km service interval validation.
- Fuel and Maintenance lists/approvals are separated even though they share BC table 50865.
- Vehicle Transfer is available in navigation.
- Gate Pass records remain separated into Store Issue, Transfer Order and Asset Transfer sections.
